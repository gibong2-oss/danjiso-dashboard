## 3. 정산현황 매출 계산 로직

### 3-1. 기본 단가

| 서비스 | 발송수단 | 단가 |
|--------|----------|------|
| 전자투표 | - | 40원/건 |
| 알림서비스 | 카카오알림톡 | 20원/건 |
| 알림서비스 | SMS | 20원/건 |
| 알림서비스 | LMS | 40원/건 |
| 알림서비스 | MMS | 80원/건 |
| 설문조사 | - | 40원/건 |

### 3-2. 제외 대상

정산 매출 계산 시 아래 조건에 해당하는 건은 **전액 제외**한다.

1. **무료단지**: 해당 월 `무료단지` 시트에 등록된 단지코드
2. **직방전환단지**: 단지코드가 `직방전환단지` 시트에 존재하고, 해당 날짜가 전환 기간(시작일~종료일) 안에 포함된 경우

### 3-3. 프로모션단지 처리

프로모션단지는 **무료 쿼터(잔여수량)** 를 초과한 발송분에 대해서만 정산한다.

```
월별 총 발송량 = Σ(해당 단지의 일별 발송건수)

정산 비율(settled_ratio) =
  - 총 발송량 ≤ 쿼터   → 0.0  (전액 무료)
  - 총 발송량 > 쿼터   → (총 발송량 - 쿼터) / 총 발송량

정산 매출 = 발송건수 × 단가 × settled_ratio
```

> 주의: `settled_ratio`는 **월 단위**로 선 계산 후, 해당 월의 모든 일별 행에 동일하게 적용한다.

### 3-4. 데이터 시작 기준

- 정산현황 데이터는 **2026-03 이후**만 유효
- `SETTLE_START = '2026-03'` 상수로 관리
- 월(YYYY-MM)이 SETTLE_START 미만인 행은 계산에서 제외

### 3-5. 불완전 첫 주 처리

주별(weekly) 집계 시, **첫 주의 월요일이 SETTLE_START 이전**이면 해당 주를 제거한다.

```
예) SETTLE_START = 2026-03-01 (일요일)
    2026-W09 = 2/23(월) ~ 3/1(일) → 3/1 하루치만 포함 → 불완전 주 → 제거
    → 주별 데이터는 2026-W10(3/2~)부터 시작
```

이는 첫 주의 WoW가 비정상적으로 높게 계산되는 문제를 방지한다.

## 5. HTML 대시보드 주요 함수

### 5-1. 공통 유틸

```javascript
// GNB 날짜 필터 객체
gFilt = { start: "YYYY-MM", end: "YYYY-MM", code: "단지코드 or null" }

// 월별 레이블 필터 (SD: Slice by Date)
SD(labels, ...vals)
// → gFilt.start ~ end 범위의 인덱스만 반환: { lbl: [], d: [[...]] }

// 일/주별 레이블 필터
sliceDateLabels(labels)
// → YYYY-MM-DD 또는 YYYY-Www 형식 레이블을 gFilt 기준으로 인덱스 배열 반환
```

### 5-2. 차트 함수

| 함수 | 용도 |
|------|------|
| `barCfg(labels, data, color, yFmt)` | 공통 bar 차트 옵션 생성 |
| `drawUsageCnt(id, color, data)` | 이용 건수 현황 (일/주/월/분기/연간) |
| `drawUsageRev(id, data_est, data_settle)` | 매출 현황 (일/주/월/분기/연간 × 건수추정/정산현황) |
| `drawGrowthCnt(id, wk_labels, wk_wow, mo_labels, mo_mom, mo_qoq, mo_yoy)` | 성장률-이용건수 기반 (WoW/MoM/QoQ/YoY) |
| `drawGrowthRev(id, revWkLabels, revWkWow, estLabels, estVals, settledMonths, settledVals, settleWkLabels, settleWkWow)` | 성장률-매출 기반 (WoW/MoM/QoQ/YoY × 건수추정/정산현황) |

### 5-3. `drawUsageRev` 데이터 구조

```javascript
// data_est: 건수추정 매출
{
  day: { labels: [], vals: [] },   // 일별
  wk:  { labels: [], vals: [] },   // 주별
  mo:  { labels: [], vals: [] }    // 월별 (분기/연간은 mo에서 집계)
}

// data_settle: 정산현황 매출 (2026-03 이후)
{
  day: { labels: [], vals: [] },
  wk:  { labels: [], vals: [] },
  mo:  { labels: [], vals: [] }
}
```

### 5-4. `drawGrowthRev` 내부 분기

```
버튼 조합         데이터 소스
──────────────────────────────────
WoW × 건수추정   revWkLabels / revWkWow
WoW × 정산현황   settleWkLabels / settleWkWow
MoM/QoQ/YoY × 건수추정  estLabels / estVals → calcGrowthRates()
MoM/QoQ/YoY × 정산현황  settledMonths / settledVals → calcGrowthRates()
```

## 6. 차트 공통 옵션 규칙

- **`clip: false`**: 데이터 레이블이 차트 영역 밖으로 나가도 잘리지 않도록 설정 (전체 bar 차트 적용)
- **`layout.padding.top: 18~22`**: 상단 레이블 공간 확보
- **`grace: "10%"`**: y축 최대값 위로 10% 여유 공간 → 최고점 레이블 잘림 방지
- **`datalabels`**: `anchor:"end", align:"end"` (bar 상단 표시), font size 9px

## 7. 단지 활성도 등급 기준

활성도는 **MAU ÷ 세대수 × 100(%)** 비율로 산정한다. 데이터 소스: `단지별 MAU` 시트 + `단지 정보` 시트 (세대수).

| 등급 | 색상 | MAU/세대수 기준 | 설명 |
|------|------|-----------------|------|
| 매우 활성화 | #52C41A (초록) | ≥ 100% | 전 세대 이상 참여 |
| 활성화 | #95DE64 (연초록) | 70% 이상 ~ 100% 미만 | |
| 사용 | #5EC8E8 (하늘) | 40% 이상 ~ 70% 미만 | |
| 비활성 | #FFC107 (노랑) | 10% 이상 ~ 40% 미만 | |
| 이탈 | #FF7043 (주황) | 0% 초과 ~ 10% 미만 | |
| 미이용 | #EF5350 (빨강) | MAU = 0 | 앱 미사용 단지 |

> 활성도 분류는 `add_settle_wk.py`가 아닌 `activity_data.json` 생성 시점에 적용된다.  
> 엑셀 원본의 `단지별 MAU` 시트에서 최신월 MAU와 `단지 정보` 시트의 세대수를 결합하여 산정한다.

## 8. 데이터 갱신 절차

원본 엑셀 파일이 업데이트되면 아래 순서로 실행한다.

```bash
# 1. 정산현황 일/주별 데이터 계산 및 growth_data.json 업데이트
python3 add_settle_wk.py

# 2. HTML 대시보드 생성
python3 gen_v4c.py
```

> `add_settle_wk.py`는 `gen_v4c.py` **이전에** 반드시 실행해야 한다.  
> `gen_v4c.py`는 `growth_data.json`을 읽어 HTML에 임베드하기 때문이다.
